Modular City

About

This project is available to test at my GitHub

This project builds a modular metropolis using Processing Java. Static layouts were an option, but I went with a randomly generated approach with a rustic vibe.
The vertex coordinates of each object had to be declared in code. In order to increase performance, I implemented render distance and wrote code that determines whether a field already has a well nearby before adding one.

Project Info

Roles: Development, Texturing

Team Size: 1 person

Time frame: 3 weeks

Year: 2024

Summary

For the layout the program starts by setting up the grid, defining which lines and columns will be roads and rivers, sets bridges at the intersections, making sure both sides are the same height, and then defines the rest of the cells, most importantly the fields and wells so that each field is necessarily next to at least 1 well.

As well ass the features mentioned above I also implemented WASD movement and a day and night cycle, this was achieved by a directional light rotates around the city with a shift in color

I also ended up making my own pixel art textures for the objects, bellow I have some comparison images between no render distance nor textures and the final version
