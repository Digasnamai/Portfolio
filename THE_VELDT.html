THE VELDT

About

This project was created on the theme of "transmediation". This text-based adventure game is based on Ray Bradbury's story "The Veldt," which is a cautionary tale about a futuristic family living in a fully automated house that caters to their every need.
The user follows the events of the story from the perspective of the house via a retro computer terminal, while the computer sends the user queries to which the user must respond in order to proceed.
In this project, I was in charge of constructing the website and all the interactions, rewriting the story, audio editing and implementation, and putting it live on the web.

Project Info

Roles: Development, Writing, Audio editing

Team Size: 2 people

Time frame: 4 weeks

Year: 2025

Summary

This project is available to test at THE VELDT

In this game the user is an employee of the company "Happy Life" browsing old files about an accident in the original story.
Throughout the game, the computer sends queries that must be replied using text inputs. The computer begins as something harmless and progresses to a sinister, nearly dangerous persona.

For the text content I had to rewrite the story from the perspective of the house, for this I followed the following rules:

- Log Encapsulation: The story should be contained within "logs" generated by the house.
- Literalism: Use descriptions that are apathetic, cold, and strictly mechanical.
- Objectivity: Scenes should be described as literal data rather than sensory experiences.
- Dehumanization: Refer to character as variables or elements within a code base using the format CLASSIFICATION[X] (e.g., HOMEOWNER[0], OFFSPRING[1]).
- Transcript Formatting: dialogue must be presented inside transcripts.
- File Access: Audio must be treated as files that require the user to play them through the terminal interface.
- Process Mapping: Physical house actions (toggling lights, device functions) must be described as "programs" or "processes" running in the system.

For the interactions and queries I developed a simple Q&A system that takes in the user input and compares it to the currently available response or interaction options, returning either an error message or a response action. These actions can range from a follow-up question to a response displayed on an overlay or responses that change after being shown, etc.
